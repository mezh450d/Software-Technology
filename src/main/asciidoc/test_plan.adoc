= Test-Plan

Dieser Testplan stellt eine vereinfachte Version der IEEE 829-1998 Vorlage dar.

== Einführung
Wozu dient das Dokument? Zielgruppe?

== Aufbau der Testbezeichner
Nach welchem Schema sind die Tests benannt?

* Testbezeichner:
** U - Unit Test, I - Integrationstest
* Verantwortlicher
** J - Jing
** M - Meng
** S - Shiyue
** Z - Ziyi
* folgendes Schema wird benutzt
** [Testbezeichner]-[Verantwortlicher]-[Klassenname]-[Methodennummer]

== Test Gegenstände

== Zu testende Funktionalitäten

- Konstruktoren auf invalite Parameter testen

- Funktionen auf invalite Parameter testen

== Nicht zu testende Funktionalitäten
z.B. Bibliotheken von Drittanbietern (eigene Tests beschränken sich auf Schnittstelle, bzw. Zusammenspiel).

== Herangehensweise

- Klassen werden mit JUnit getestet.
